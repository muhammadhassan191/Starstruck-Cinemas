The Core Technology Stack

Frontend & Layout

Mobile & Persistence

Powered by Capacitor v6 embedded Native WebView bridges, communicating natively with a Local SQLite3 Serverless database inside the OS.

Why SQLite instead of Firebase?

Traditional (Firebase/Mongo) Our Approach (SQLite)
Requires mandatory internet connection ➔ Offline First, C-language file access
Introduces remote API latency (200ms+) ➔ Local device processing (< 12ms)
Cloud schema structure ➔ Self-Contained Relational Execution Engine
Database Architecture

Geographic Database Distribution

Engineered a Multi-Cinema Schema allowing 5 global locations (New York, LA, Queens, Bronx, Brooklyn). Strict relational mapping forces movies to filter by 'cinema_location'.

The PKR Loyalty Trigger

We dynamically altered the monetary logic to Pakistani Rupees (Rs. 1000 - Rs. 3000).

When an SQL `INSERT` is fired at checkout, Middleware calculates a flat 10% mathematical deduction and fires a simultaneous `UPDATE` to the user's loyalty trace array.

Mobile Frameworks

Why Capacitor over React Native?

The WebView Injection

Unlike React Native which translates UI to widgets, Capacitor embeds our Next.js payload inside a blazing-fast native Android WebView container allowing flawless UI control.

Bi-Directional Bridges

We didn't just wrap a webpage. We introduced Javascript-to-Java bridges, meaning our web code directly commands Android FileSystem and Share Trays via OS Intents.

@capacitor/filesystem

Allows the JS Payload to securely write encrypted Blob files directly into the Android hardware's Local Cache Directory tree without triggering security flags.

Native Hardware Plugins

By executing Android SDK plugins via NPM, we shattered the typical web-browser permissions model entirely to allow deep hardware communication.

Artificial Intelligence

Zero External API Dependencies

Did we just plug in ChatGPT to act as a movie chatbot? No. To guarantee zero latency offline availability, we wrote a custom engine.

O(N * T) Token Analysis

The engine physically slices incoming user strings into localized array tokens. It drops Stop Words mathematically and recursively matches Intent parameters across the SQLite Schema.

RegEx Pattern Matching

Uses logical inference mapping ('time', 'price', 'review'). If a user says "Review matrix bad", it performs database updates entirely computationally.

Android Intent API

Native Implicit File Sharing

Standard web `window.print()` throws corrupt browser printing interfaces. We bypassed this fully.


We trigger an "Implicit Intent" (`ACTION_SEND`) passing the raw physical URI of a file. This elegantly forces the Android Operating System's native file popups to execute instead of Chrome.

Document Output

  • Compiled purely via `html2pdf.js` Blob structures.
  • Translates localized DIV blocks to binary Base64.
  • Saves to Google Drive / local Android files.
Algorithms

Breadth-First Search (BFS) Injection

UI block-selection is an algorithmic puzzle. Groups need to sit cohesively without fragmenting.

Spatial Auto-Grouping

We introduced a BFS scanning architecture over the available seating arrays. When a user requests '3' seats, the engine computationally traces the nearest non-broken column block to preserve family structure.
